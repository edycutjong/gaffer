## Rule 4 — on-device QVAC compliance (track-specific finding)

**Verdict: compliant, with the honesty caveat already disclosed.**

- The only inference/speech path that exists is `lib/engines/qvac.js`, whose entire surface
  is `@qvac/sdk` calls (`loadModel`, `completion({stream:true})` → `tokenStream`,
  `textToSpeech`, `startQVACProvider`) — grep of the codebase finds zero references to
  `openai`, `anthropic`, or any HTTP call to a third-party inference endpoint.
- `lib/engines/sim.js` is a **local deterministic grammar** (no network call at all, not a
  disguised cloud call) used only when the SDK/model is absent — this is the correct shape
  for Rule 4 (a disclosed local fallback, not a cloud substitute) and is labelled everywhere.
- `scripts/verify_offline.js` is a structural enforcement mechanism, not just a doc claim: it
  actively blocks non-local sockets and runs the full commentary + P2P-offload loop under that
  guard, in CI on every push (`.github/workflows/ci.yml`, "offline-proof" stage). Re-run live
  in this audit: passes.
- **Residual honesty note, already in `docs/AUDIT_REPORT.md` (R3):** the guard covers
  TCP/TLS/HTTP(S)/fetch; HyperDHT itself uses UDP, so a hypothetical UDP exfiltration path
  would not be caught by the guard. No such path exists in the dependency tree (pure Hyperswarm
  stack), but a judge auditing Rule 4 mechanically (not just reading the disclosure) should
  know the guard's boundary.
- The P2P delegation (`startQVACProvider` / Gaffer's own provider protocol) keeps all
  inference **on a peer device**, never a cloud host — satisfying the track's spirit (device
  compute lending) as well as its letter.
